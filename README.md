# Obsidian Vault Manager Skill

> A [Claude Code](https://www.anthropic.com/claude-code) skill for managing large Obsidian vaults with **4+1 layer info-stratification + PARA + P.A.I.R + Cold Storage** architecture.
>
> 一个 Claude Code Skill,用 **4+1 层信息分层 + PARA + P.A.I.R + 冷藏区隔离 + 命名前缀规范 + 高频场景路由** 六套机制,让上千文件的 Obsidian 知识库保持高信噪比,**让 AI 时代最稀缺的"个人经验"不被 SOP 流程稀释**。

---

## 这个 Skill 解决什么问题

当 Obsidian 知识库膨胀到 5000+ 文件后,会出现典型的"知识沼泽"症状:

- 每天自动抓取的推特/公众号原文淹没真正有用的精炼笔记
- 主题分散在十几个目录里,找一个东西要打开 N 个文件夹
- 概念笔记重复,比如"知识库管理"建了三个节点
- 双向链接(Wiki Link)失修,无法把相关知识串起来
- 检索时大量噪声进入上下文,让 AI 助手回答质量下降
- **(v3 新发现)个人原话/亲历/感受被 SOP 萃取流程"磨平",失去 AI 时代最稀缺的"你"**

**这个 Skill 通过六个核心机制解决以上问题**:

1. **4+1 层信息分层(v3)**:个人经验 L1 / 在场素材 L2 / 入口 L3 / 萃取成品 L4 / 冷藏 L5,物理隔离
2. **冷藏区隔离**:把高频低密度信息(自动抓取类)放 `99-冷藏/`,主区检索默认豁免
3. **命名前缀规范**:`[MOC]` / `[理论]` / `[工程]` / `[方法论]` 让概念节点一眼分类
4. **淘金机制**:从冷藏区"复制"(不剪切)到主区,源文件加 `cherry_picked` 标记
5. **高频场景路由表(v3)**:12 个高频场景 → 对应目录,让 AI 助手自动路由
6. **L1 个人经验保鲜(v3)**:原话/感受/亲历独立目录,绝不混入 SOP 流程

### 4+1 层信息分层模型(v3 核心)

| 层 | 名称 | 定义 | 物理目录 | frontmatter |
|---|------|-----|---------|-------------|
| **L1** | 个人经验层 | 用户原话、亲历、一手感受 | `02-Areas/个人经验/` | `layer: experience` |
| **L2** | 在场素材层 | 用户在场录音/会议原文 | `07-Archive/播客转录/` | `layer: scene` |
| **L3** | 外部入口层 | 待分类原料 | `00-Inbox/` | `layer: inbox` |
| **L4** | 萃取成品层 | 加工后的 SOP / 概念 / 方法论 | `02-Areas/04/05/06` | `layer: extract` |
| **L5** | 冷藏原料层 | 自动抓取的低频原料 | `99-冷藏/` | `layer: cold` |

**关键洞察**:AI 上下文越来越大不代表"不用整理了",真正稀缺的是上下文**质量**。L1 个人经验是 AI 永远学不会的"你",必须用目录物理隔离,绝不让"萃取冲动"污染原汁原味的一手感受。

### 冷藏区(Cold Storage)隔离原则

**问题**:自动抓取的推特/公众号每天产出几十到几百篇原文,稀释主区信噪比。

**方案**:物理隔离 + 检索豁免 + 淘金机制。

```bash
# 主区检索(默认豁免冷藏)
grep -r "QUERY" <vault-root>/ \
  --exclude-dir=99-冷藏 \
  --exclude-dir=.obsidian

# 淘金检索(显式声明)
grep -r "QUERY" <vault-root>/99-冷藏

# L1 个人经验专项检索(v3 新增)
grep -r "QUERY" <vault-root>/02-Areas/个人经验/
```

### 命名前缀规范(04-Concepts)

| 前缀 | 含义 | 示例 |
|------|------|------|
| `[MOC]` | Map of Content,多文件汇总索引 | `[MOC] 用户分层与定价体系.md` |
| `[理论]` | 概念定义、原理、本质 | `[理论] 知识库与RAG完全指南.md` |
| `[工程]` | 具体实现、技巧、最佳实践 | `[工程] 稀缺性推理在上下文工程中的应用.md` |
| `[方法论]` | SOP 之上的元方法论 | `[方法论] 知识库管理.md` |

**冗余词黑名单**(不得用于节点名):调研、报告、深度、完全、详解、全景

**英文节点**:除专有名词(NVIDIA / API / Harness / Hermes / Karpathy)外,全部翻译成中文,专有名词附中文说明。

### L1 个人经验层细分(v3 新增)

| 子目录 | 准入规则 | event_type |
|--------|---------|------------|
| `原话与决策/` | 用户亲口说出的话、重大决策原话、对外正式表态 | `原话` |

| 目录 | 定位 | 收录什么 |

| `99-冷藏/朋友圈素材/` | 低频原料(L5) | 社交媒体素材 |

## 工作流程

### 收录流程

```
内容 → 类型判断(参照 4+1 层 + 高频场景路由) → 选目录 → 创建笔记 → 更新 MOC
```

**Step 1:类型判断 → 目录路由**

```
内容是什么?
├── 用户自己的原话/感受/亲历 → 02-Areas/个人经验/<子目录>/(L1,v3)
├── 用户在场录音/会议原文 → 07-Archive/播客转录/(L2)
├── 具体项目交付物 → 01-Projects/(L4)
├── 某领域的深度内容 → 02-Areas/<领域>/(L4)
├── 关于某个人的深度拆解 → 03-People/(L4)
├── 可跨域复用的核心概念 → 04-Concepts/(L4,必带 [前缀])
├── 可复用的方法论/SOP → 05-Playbooks/(L4)
├── 工具/资源/失败案例 → 06-Library/<子目录>/(L4)
├── 已结案项目 / 高价值原料 → 07-Archive/<子目录>/
├── 推特/公众号/朋友圈/活动文案 → 99-冷藏/<子目录>/(L5)
└── 不确定 → 00-Inbox/(L3)
```

**Step 2:创建笔记**

通用模板:

```markdown
---
tags: [主标签, 辅助标签]
created: YYYY-MM-DD
source: <来源URL或说明>
author: <作者>
layer: extract  # v3 必填
cherry_picked: false
---

# 标题

> 一句话摘要

## 正文内容

(原文完整保留)

## AI 助手分析与梳理

(助手的补充分析,与原文明确分隔)

## 相关链接

- [[相关概念1]]
- [[相关文档2]]
```

L1 个人经验层模板(v3 新增):

```markdown
---
tags: [个人经验, 主标签]
created: YYYY-MM-DD
layer: experience
event_date: YYYY-MM-DD
event_type: 原话 / 感受 / 亲历
---

# <事件标题>

> 一句话上下文

## 原话/感受/亲历记录

(原汁原味,不萃取,不润色)
```

**Step 3:建立 Wiki Link**

每篇精炼笔记必须建立三类链接:

1. **向上链接**:链接到所属领域的 MOC(`_MOC-*.md`)
2. **横向链接**:链接到相关概念节点(`04-Concepts/`)
3. **向下链接**:链接到引用的具体文档

### 检索流程(三步法)

1. **目录路由**:根据问题主题匹配目录(参考高频场景路由表)
2. **关键词 Grep**:在匹配目录内搜,默认豁免冷藏
3. **Wiki Link 跳转**:读取 `[[]]` 链接追溯关联知识

### 淘金流程(冷藏区 → 主区)

**触发时机**:助手判断主区无果但冷藏区可能有,或用户明确说"翻一下推特/公众号"。

**3 步 SOP**:

1. **定位**:在 `99-冷藏/` 内显式 grep 关键词
2. **评估**:核对 frontmatter 的 source/author/date
3. **迁移**:**复制**(不是 mv)到 02-Areas + 原文件加 `cherry_picked: true`

### 整理流程

| 频率 | 任务 |
|------|------|
| 每周 | 清理 `00-Inbox/`(归位 → 移动 → 更新 MOC) |
| 每季度 | 冷藏区淘金 |
| 持续 | Archive 提炼(高价值原料 → 02-Areas/05-Playbooks 精炼笔记) |

## 禁止事项

1. **不删除任何文件**(除非用户明确要求)
2. **不修改原文内容**(只追加,原文一字不改)
3. **不用 Write 覆盖已有文件**(用 Edit)
4. **不改朋友圈素材的序号和内容**
5. **不把敏感信息写入 Vault**(API key / 凭证 / 密钥保留在本地配置文件)
6. **不批量从冷藏区往主区搬**(违背冷藏初衷)
7. **不在主区写入低频原料**(推特/公众号自动抓取必须走冷藏区)
8. **不创建无前缀的 04-Concepts 节点**
9. **不在 `02-Areas/个人经验/` 用 SOP 化格式写**(v3)
10. **frontmatter 必须含 layer 字段**(v3)

## 安装与使用

### 1. 安装

把 `SKILL.md` 放到 Claude Code 的 skills 目录:

```bash
mkdir -p ~/.claude/skills/obsidian-vault-manager
cp SKILL.md ~/.claude/skills/obsidian-vault-manager/
cp PHILOSOPHY.md ~/.claude/skills/obsidian-vault-manager/  # v3 新增,可选
```

### 2. 配置

修改 `SKILL.md` 中的 `Vault 路径` 占位符,改成你自己的 Obsidian Vault 实际路径:

```
<vault-root>/  →  /path/to/your/obsidian/vault/
```

### 3. 触发

在 Claude Code 中说以下任一句子,Claude 会自动加载本 skill:

- 「收录到知识库」
- 「存到 Obsidian」
- 「整理笔记」
- 「翻一下推特/公众号有没有 XX」(触发淘金)
- 「补充 Wiki Link」
- 「我刚跟 X 说……」/「我感觉……」/「今天发生了……」(v3,自动走 L1)

### 4. 自定义

按你自己的领域调整 `02-Areas/` 子目录:

```
02-Areas/
├── 个人经验/                  # v3 新增,L1
│   ├── 原话与决策/
│   ├── 感受与判断/
│   └── 亲历事件/
├── 你的领域 A/
├── 你的领域 B/
└── ...
```

`04-Concepts/` 节点按 `[前缀]` 规范创建即可。

## 适用场景

- 上千文件的 Obsidian Vault 维护者
- 有大量自动抓取流(推特/公众号/RSS)淹没主区的用户
- 想给 AI 助手提供高信噪比检索语料的用户
- 已有零散 PARA 实践但缺统一收录/检索 SOP 的用户
- **重视个人经验保鲜的用户(v3)**——不希望"萃取冲动"磨平一手感受

## 不适用场景

- 文件数 < 200 的小型 Vault(过度工程)
- 只用 Obsidian 做日记/笔记,无自动抓取流的用户
- 不使用 Claude Code 或类似 AI 助手的用户
- 不在乎个人经验"原汁原味"的用户(L1 机制对你没价值)
